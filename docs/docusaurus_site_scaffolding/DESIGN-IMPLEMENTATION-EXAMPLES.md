### UNIMARC Navigation Strategy

Given the large number of element sets, UNIMARC requires enhanced navigation:

```tsx
// src/components/UNIMARCNavigator/UNIMARCNavigator.tsx
import React, { useState } from 'react';
import { TreeView, TreeItem } from '@mui/lab';
import { ExpandMore, ChevronRight } from '@mui/icons-material';
import styles from './UNIMARCNavigator.module.css';

const fieldGroups = [
  {
    id: 'control',
    label: 'Control & Coded Information',
    sets: ['0XX', '1XX']
  },
  {
    id: 'descriptive',
    label: 'Descriptive Information',
    sets: ['2XX', '3XX']
  },
  {
    id: 'linking',
    label: 'Linking Fields',
    sets: ['41X', '42X', '43X', '44X', '45X', '46X', '47X', '48X']
  },
  {
    id: 'titles',
    label: 'Related Titles',
    sets: ['5XX']
  },
  {
    id: 'subject',
    label: 'Subject Access',
    sets: ['60X', '61X', '62X', '66X', '67X', '68X']
  },
  {
    id: 'responsibility',
    label: 'Responsibility & Notes',
    sets: ['7XX', '801', '802', '830', '850', '856', '886']
  }
];

export function UNIMARCNavigator({ currentSet }) {
  const [expanded, setExpanded] = useState<string[]>(['root']);

  return (
    <div className={styles.navigator}>
      <h3>UNIMARC Element Sets</h3>
      <TreeView
        defaultCollapseIcon={<ExpandMore />}
        defaultExpandIcon={<ChevronRight />}
        expanded={expanded}
        onNodeToggle={(event, nodeIds) => setExpanded(nodeIds)}
      >
        {fieldGroups.map(group => (
          <TreeItem key={group.id} nodeId={group.id} label={group.label}>
            {group.sets.map(setId => (
              <TreeItem
                key={setId}
                nodeId={setId}
                label={`${setId} Fields`}
                className={currentSet === setId ? styles.current : ''}
                onClick={() => window.location.href = `/docs/elements/${setId}`}
              />
            ))}
          </TreeItem>
        ))}
      </TreeView>
    </div>
  );
}
```

## Key Implementation Patterns

### 1. Scalable Navigation
- For 2-3 element sets (ISBD): Simple dropdown/tabs
- For 5-10 element sets (ISBD-LRM): Categorized list
- For 20+ element sets (UNIMARC): Hierarchical tree with grouping

### 2. Consistent URLs
```
/docs/elements/{element-set-id}/{category}/{element-id}
/docs/vocabularies/{vocabulary-id}/{term-id}
```

### 3. Cross-References
Elements can reference other elements using:
- Same element set: `[has title proper](../P1001)`
- Different element set: `[unconstrained version](/docs/elements/unconstrained/elements/U1001)`
- Vocabulary term: `[cartographic](/docs/vocabularies/content-form/T1001)`
